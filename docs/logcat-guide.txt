#LOGCAT GUIDE:

# what is logcat?
- logcat is a command line tool that show system and application logs from an android device.

# Why it is important?
- helps detect the crashes
- helps debug failures
- used during automation failure analysis


- LOGGING TOOL ? 
: its a tool to tack all activity running in a app
: catch errors  
:help in debugging
:detect the reason of app crash  and by the help of this developer can solve the error 


Q: HOW LOGCAT WORKS ?
- The Android system and the application generate logs during app execution. Logcat collects these logs and displays them on the screen in real time as messages.

ERROR VS CRASH ?
ERROR (E)= only error signed as E .. only error but app is running ..error can be network error 
CRASH= app stop working , logcat shows FATAL EXCEPTION 

STEPS TO FIND CRASH :
• first clear previous logs if opend 
• first find your app package name 
• after this , open app and write commands 
• first find process id of that perticular running app  using adb shell pidof pkg_name 
• after this copy that pid and do filter using adb logcat pid=id 
• now only that app logs display .
• if there is an error in red color ..like FATAL ERROR 
• then you successfully find the crash 
• now carefully read the error on which thread ..ex main or else 
• read the process line .. 
• now stack tracing ..  read.. which  file/class .. exception type .. line no (location) generate error
